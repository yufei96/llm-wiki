## Key Findings

### MOSA Definition (Section 1.1)
MOSA = modular design + open standards consensus compliance + system integration with modular architecture. Covers technical AND business considerations.

### 5 MOSA Pillars (Section 2.1)
1. **Establish an Enabling Environment** (requirements, policies, governance)
2. **Employ a Modular Design** (functional decomposition, component isolation)
3. **Select Appropriate Open Standards** (standards selection, conformance)
4. **Designate Key Interfaces** (open, published interfaces)
5. **Ensure Compliance** (verification & validation, conformance testing)

### Program Document Requirements (Section 3.1.5)
MOSA must be addressed in:
- **Acquisition Strategy** (DoDI 5000.85, Para 3C.3.a.(5)) — MOSA plan, technical considerations, major system components
- **Systems Engineering Plan** (DoDI 5000.88, Para 3.4.a) — MOSA technical approach, architecture description
- **Product Support Strategy** (10 U.S.C. 4324)
- **Life Cycle Sustainment Plan** (LCSP)
- **Capability Development Document** (CDD) & **Initial Capabilities Document** (ICD)
- **Requests for Proposals (RFPs)**

### Cost/Benefit Analysis (Section 3.0)
MOSA should be evaluated for cost-benefit impact. Programs must justify if MOSA is NOT feasible or cost-effective.
