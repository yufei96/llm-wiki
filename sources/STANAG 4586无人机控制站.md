## 概述

本文为北约标准化协议STANAG 4586的技术介绍文献，原文发表于北约科技组织(STO)教育笔记STO-EN-SCI-271（第3章），作者Mário Monteiro Marques（葡萄牙海军学校）。STANAG 4586旨在规范无人机控制站(UCS)接口以实现北约无人机系统互操作性，定义了5级互操作性(LOI)和UCS功能架构。除STANAG 4586外，文档还综述了与无人机相关的其他北约STANAG标准（7085、7023、4545、4607、4609、7024、4575）。

## 核心内容

### 背景与动机

1990年代初的北约互操作性设计研究表明，强制所有成员国采购相同系统不可行。与其让单一承包商垄断北约侦察系统，不如借鉴计算机通信模式——通过精确定义接口实现数据交换的成功。

随着无人机种类和供应商日益增多，异构系统之间的互操作性难度增大。无人机已被证明是帮助联合部队指挥官实现战区、作战和战术目标的宝贵资产，但其协同需要：快速任务分配、互相控制无人机及其载荷、以及快速分发生成的信息。

### STANAG 4586演进

- 1998年北约专家组开始制定STANAG 4586
- **第1版**：2003年完成，2004年获成员国批准
- **第2版**：2007年发布
- **第3版**：2012年发布（当前版本）

STANAG 4586分为两个附件：附件A提供术语表，附件B提供通信架构概述（由三个附录支撑：B1数据链接口、B2指挥控制接口、B3人机接口）。

### 目标

规定应实施的接口以达到特定概念作战(CONOPS)所需的互操作性等级(LOI)，通过实施标准UCS接口实现：
- 与不同无人机及其载荷通信
- 与不同的C4I系统通信
- 促进不同来源组件的集成
- 传统系统互操作性

合规无人机将获认证，通过资产共享增强北约联合灵活性。

### 五级互操作性(LOI)

| 级别 | 描述 |
|---|---|
| **LOI 1** | 间接接收和/或发射传感器的产品数据和元数据 |
| **LOI 2** | 直接从无人机接收传感器产品数据和元数据 |
| **LOI 3** | 无人机载荷的控制和监视（除非指定仅监视） |
| **LOI 4** | 无人机控制与监视（发射回收除外，除非指定仅监视） |
| **LOI 5** | 无人机发射回收的控制与监视（除非指定仅监视）|

LOI 2监视受载荷类型和数量限制。LOI 3受载荷类型、数量和格式限制。LOI 4不受载荷影响。

### UCS功能架构

架构定义以下元素和接口：

1. **空中飞行器(AV)**：无人机平台本体
2. **飞行器专用模块(VSM)**：提供无人机专有的通信协议/接口时序/数据格式，担任CUCS标准消息与AV之间的翻译。通常由无人机制造商提供。若数据链本身STANAG 4586兼容，则VSM非必需。
   - 功能：消息翻译、数据打包/解包、数据库管理、数据链消息接口管理、发射回收接口管理、传感器数据A/D转换
3. **数据链接口(DLI)**：CUCS与VSM之间的接口，定义飞行器和载荷独立的控制和状态消息处理机制
4. **核心UCS(CUCS)**：提供用户界面以执行无人机任务所有阶段。功能包括：载荷数据接收/处理/分发、任务规划、飞行器/载荷/数据链的监视与控制、支持未来扩展能力、可承载VSM和CCISM功能
5. **指挥控制接口(CCI)**：定义与C4I系统之间的标准消息集和协议（C4I系统/节点独立）。覆盖所有任务阶段：
   - 飞行前：任务分配、战术态势、环境数据、任务约束
   - 飞行中：状态/服务消息、载荷数据、进展报告
   - 飞行后：状态/服务消息、载荷数据、后期利用报告、任务报告
   - 网络和通信应遵循NC3TA(NATO C3技术架构)
6. **人机接口(HCI)**：规定CUCS应支持的操作员显示和输入要求，但不强制人因工程设计（如显示器数量、手动控制等）
7. **指挥控制接口专用模块(CCISM)**：功能类似VSM，封装CCI数据并提供与传统C4I系统的翻译。当需要与不直接兼容STANAG 4586的传统系统通信时使用

### STANAG 4586的局限与挑战

1. **基于消息的标准**：最初为长航时无人机要求开发，载荷相关消息可被消除或修改
2. **仅限北约成员国**：北约"非保密"级别可能成为非北约国家全球供应商的障碍（如日本机器人产业）
3. **军事标准导向**：合规要求高，认证费用昂贵（如STANAG 7085），对小平台供应商构成进入壁垒
4. **未定义通用操作环境(COE)**：仅指定操作系统须支持/集成指定网络/传输协议和用户应用
5. **未考虑动态网络拓扑**：不处理通信不可靠的场景和由通信约束决定的飞行器行为
6. **虽然不提供完整互操作性解决方案**，但STANAG 4586是朝着该方向迈出的关键一步，为未来发展提供了路线图

## 笔记

1. STANAG 4586 Edition 3 (2012)为当前最新版本，核心在于通过接口标准化实现异构无人机系统间的互操作。
2. LOI 4和LOI 5是关键区分——LOI 4允许控制和监视无人机（不含发射回收），LOI 5才允许完整的发射回收控制。
3. **VSM是关键架构要素**：将厂商专有协议翻译为标准接口，是实现即插即用的核心；如数据链原生支持STANAG 4586则可省略VSM。
4. **与美军标准的关联**：STANAG 7085 Annex B的Implementation 1描述了美军通用数据链(CDL)，显示北约标准与美国标准之间的映射关系。
5. **STANAG生态系统**：STANAG 4586定义了控制接口架构，而7085/7023/4545/4607/4609/7024/4575等处理数据层面的互操作性——它们共同构成完整的无人机互操作性架构。
6. 合规认证虽然昂贵，但可以提前为可能的组合操作做预先认证，长远降低操作风险。
7. 源文件：Marques, M.M., "STANAG 4586 – Standard Interfaces of UAV Control System (UCS) for NATO UAV Interoperability", STO-EN-SCI-271, Chapter 3, pp. 3-1 to 3-14.
