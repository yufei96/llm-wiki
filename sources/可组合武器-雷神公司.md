## 概述

雷神（Raytheon，现RTX）先进技术团队提出的可组合武器（Composable Weapons）方法，是一种革命性的导弹开发范式。核心理念是使用可扩展、可复用的基础组件，通过通用接口快速组合新武器系统，以"相关性速度"交付新能力。技术总监John C. Otto将这一方法描述为"一个子系统工具箱，可以一次又一次地使用通用接口将其组合在一起"。

## 核心内容

### 三大核心原则

1. **模块化与开放性**：使用经过验证的、可负担的模块化硬件，定义开放接口以促进模块化和互操作性，支持美国政府开放系统架构愿景。

2. **软件定义**：通过软件修改实现快速能力升级，适应不同任务需求，无需物理硬件变更即可扩展武器功能。

3. **标准化工程**：使用基于模型的系统工程（MBSE），创建跨项目共享的集成系统模型，确保各武器项目对子系统理解一致。

### 九类标准化子系统组件

雷神识别并标准化了以下可用于不同武器重新配置的关键元素：

| 组件 | 功能描述 |
|------|------|
| **机体（Airframes）** | 提供气动外壳和结构支撑 |
| **航电（Avionics）** | 管理系统飞行控制、导航和任务管理 |
| **执行机构（Actuators）** | 控制气动面实现机动 |
| **传感器（Sensors）** | 精确探测、识别和跟踪目标 |
| **软件（Software）** | 集成和操作所有子系统，实现任务逻辑 |
| **推进（Propulsion）** | 提供推力将系统送达目标区域 |
| **电源系统（Power Systems）** | 调节功率分配和管理电子设备供电 |
| **有效载荷（Payloads）** | 提供所需终端效果（毁伤、干扰、侦察等） |
| **通信（Communication）** | 实现数据链更新、协同和目标信息共享 |

### 战略背景

可组合武器和软件定义孔径正在响应防御系统的新需求——系统需要更快、更智能，并在由高超音速武器和无人机群重塑的战场上保持领先。该方法直接支持国防部通过模块化开放系统方法实现竞争、创新和快速技术插入的战略目标。

## 笔记

可组合武器是WOSA原则在工业界最明确、最系统的实践表达。与普渡大学的柔性武器学术研究和AFRL的灵活武器概念相比，雷神的方法更强调标准化工程过程和MBSE驱动的跨项目系统模型共享。九类子系统组件的分类法与WOSA参考架构中的功能域划分存在映射关系。值得注意的是，雷神将"软件"作为独立的标准化组件维度，反映了软件定义武器能力在现代导弹系统中的核心地位。这一方法正在影响美军下一代导弹（如JATM、LRASM、PrSM等）的开发策略。
