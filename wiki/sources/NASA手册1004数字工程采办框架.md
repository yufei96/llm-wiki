# NASA-HDBK-1004: NASA数字工程采办框架手册

## 摘要

NASA-HDBK-1004（2020年4月1日批准，Baseline版）是NASA数字工程采办框架的核心实施文档，217页，由NASA首席工程师办公室发布。该手册提供数字工程环境（DEE）中数据获取的合同语言、数据需求描述（DRD）模板、模型基数据定义、数字数据协作、架构、互操作性标准等全方位指导。它取代了NASA-HDBK-0008（产品数据与生命周期管理手册），标志着NASA从PDLM向全面数字化采办框架的跃迁。

## 要点

### 核心定位
- **目的**：建立NASA的数字工程采办框架，包含SOW（工作说明书）合同语言和DRD，以支持数字工程环境
- **适用范围**：NASA空间飞行单项目和紧密耦合项目（NPR 7120.5），推荐用于所有其他项目
- **历史意义**：从文档驱动转向模型驱动的采办框架——"数字化模型基采办框架取代了大量传统的文档中心方法"

### 数字工程环境（DEE）定义
DEE是"将数字项目和产品技术数据在参与者之间进行协作的环境"，核心特征：
- **权威分散数据源**：去中心化但权威的数据管理
- **跨组织跨学科协作**：支持全生命周期活动（Pre-Phase A到Phase F）
- **数字模型承载信息**：取代文档，满足项目决策和信息需求
- **MBx系列**：MBD（模型基定义）、MBA（模型基分析）、MBSE、MBE（模型基企业）、PDLM

### DRD体系（附录A，核心内容）
手册提供15个标准化DRD模板：

| DRD编号 | 名称 | 用途 |
|---------|------|------|
| STD-TDP | 技术数据包 | 产品定义数据集合 |
| STD-MBx | 工程MBx模型及相关数据 | 模型基工程数据交付 |
| STD-CDMPA | 配置和数据管理过程审计 | 供应商CM合规性验证 |
| STD-CDMP | 配置和数据管理计划 | CDM实施方法描述 |
| STD-CSA | 配置状态核算 | 配置基线跟踪与报告 |
| STD-SDT | 规格和图纸树 | 产品结构文档 |
| STD-EMDAL | 工程模型、图纸及相关清单 | 设计数据交付 |
| STD-CR | 变更请求 | 工程变更管理 |
| STD-FCA/PCA | 功能/物理配置审计文档 | 审计证据交付 |
| STD-REF | 需求交换格式 | 需求数据标准化格式 |
| STD-MRI | 主记录索引 | 数据交付跟踪 |

### 数据获取合同语言（附录B）
- 100+页的标准合同语言模板
- 可直接嵌入SOW和PWS（工作性能声明）
- 覆盖数据类别指定、数据类型、特定合同交付要求
- **数据类别设计**：Type 1（承包商开发数据）/Type 2（现有数据/软件）

### 模型基数据定义（附录C）
覆盖所有关键工程数据的模型化定义：
- **系统模型（MBSE）**：SysML/其他建模语言的系统架构模型
- **CAD数据**：MCAD/ECAD/CAE/CAM
- **PMI**：制造信息（GD&T等）
- **建模与仿真**：与HDBK-7009A对齐的M&S数据
- **文档管理**：数字文档管理
- **PLM文档**：可移植产品生命周期管理文档
- **配置与数据管理**：CM流程的数字化
- **BOM产品分解结构**：物料清单的模型化
- **工程发布与变更管理**：数字化工签流程
- **零件与库管理**：零件库数字化管理

### 数字数据协作（附录D）
- **数据完整性与安全**：协作环境中的数据保护
- **DRD数字数据标准化**：数据格式、交换协议
- **数字协作框架与环境**：多组织协作的技术架构
- **协作规划与开发**：协作策略制定

### 架构（附录E）
- **模型驱动架构（MDA）**：OMG的MDA框架在NASA的适配
- **安全架构**：数据安全框架
- **信息支持系统架构（ISSA）**：IT基础设施
- **数据架构**：数据模型、数据流
- **流程架构**：业务流程数字化

### 互操作性标准（附录F）
- **模型交换与分发**：跨工具/跨组织的模型共享
- **MBE标准元素**：ISO STEP、OMG SysML、DMRL等
- **行业数据互操作性标准**：STEP AP242、JT、3D PDF、IFC等
- **互操作性与可持续性**：格式成熟度等级（9级）
- **常见互操作性问题**：格式不兼容、版本不匹配等

### 使用案例（附录G）
- MBE元素和互操作性要求的使用案例
- 为项目定义互操作性需求提供模板

### MBE计划（附录J）
- MBE计划开发指南
- MBE计划开发流程
- **MBE计划是DEE在项目层面的落地文件**

## 与MOSA/数字工程的关系

NASA-HDBK-1004是NASA数字工程采办的**合同执行层**，与DoD的[[数字工程生态]]形成直接对照：

| DoD DEE | NASA DEE |
|---------|----------|
| DoDI 5000.97（九大要素） | NASA-HDBK-1004（DRD+合同语言） |
| MOSA作为第九要素 | MBx作为核心数据交付形式 |
| 合同语言在DFARS | 合同语言在HDBK-1004附录B |
| 数字线程是概念 | 数字数据协作是实践 |

- DRD模板体系可直接用于MOSA接口数据交付
- MBE计划方法适用于[[模块化架构模式]]的实施管理
- 互操作性标准章节为MOSA接口标准化提供NASA视角
