# NASA-HDBK-7009A: 建模与仿真手册

## 摘要

NASA-HDBK-7009A（2019年5月8日批准）是NASA技术手册，为NASA-STD-7009A（建模与仿真标准）提供实施指南。该手册157页，覆盖M&S全生命周期管理，包括模型发起、概念开发、设计、构建、测试与发布、使用、以及归档各阶段的具体操作指导。它是NASA所有M&S活动的权威参考，适用于任何类型的M&S，不论学科或应用领域。

## 要点

### 文档定位
- **独特性**：在全球M&S标准中独树一帜——适用于所有类型的M&S和所有开发/使用阶段，而非单一学科或单一阶段
- **核心关注**：M&S基于分析的结果及其**报告**——为项目管理提供理解结果和评估分析可信度的通用框架
- **历史**：Baseline版2008年7月首次批准，2019年5月Revision A与NASA-STD-7009A对齐
- **前版**：取代NASA-HDBK-7009（Baseline版）

### M&S生命周期（6阶段）
手册定义了完整的M&S生命周期，与NASA项目生命周期对齐：

| 阶段 | NASA项目阶段 | 核心活动 |
|------|------------|---------|
| **模型发起** | Pre-Phase A | RWS信息收集、M&S预期用途声明、关键性评估、M&S方法论证 |
| **概念开发** | Phase A | RWS规范与数据获取、概念权衡研究、初步需求与规格 |
| **模型设计** | Phase B | 模型设计实现、概念（设计）验证 |
| **模型构建** | Phase C | 模型构建实现 |
| **测试与发布** | Phase D | 验证（Verification）、经验验证（Validation）、模型发布 |
| **模型使用** | Phase E | 预使用准备、使用评估、输入场景定义、设置与执行、灵敏度研究、数据分析、结果评估与报告 |
| **归档** | Phase F | 模型与分析归存 |

### 关键概念

#### 验证 vs 确认（V&V）
- **验证（Verification）**：确认模型正确实现了其规格——"我们是否正确地构建了模型？"
- **确认（Validation, 原文为Empirical Validation）**：通过经验数据确认模型正确代表了现实世界系统——"我们是否构建了正确的模型？"
- **置信度评估**：超越V&V，覆盖所有影响M&S结果可信度的因素

#### 工作表（Worksheet）
- **M&S生命周期工作表**：结合工作表和检查清单的功能
- 覆盖M&S规划、开发和使用三个部分
- 目的：确保M&S活动有序、一致、完整地执行
- 参考：NASA航空安全咨询小组（ASAP）报告要求M&S风险管理

#### M&S风险评估
- 附录E专门讨论M&S风险评估
- 与NASA的风险知情决策制定（RIDM）和持续风险管理（CRM）过程对齐
- 关注M&S结果的不确定性量化

### 置信度评估因素（附录D）
超越传统的V&V，覆盖所有影响结果可信度的因素：
- 模型代码验证
- 验证数据质量
- 模型使用适用性
- 不确定性量化
- 模型独立性（避免循环论证）

### 实施路径发现
NASA试点项目（Orion服务模块储罐晃动模型、Orion乘员舱着水模型、Ares推力振荡模型、MSL动力下降模型）的共同发现：
- 需要结构化的过程
- 使用一致的术语
- 超越V&V全面评估M&S可信度
- 理解RWS项目对M&S的要求
- 定义精度要求以适当地验证关键模型
- 理解如何改进M&S验证
- 处理M&S运行极限
- 提供耦合模型指导

### 与其他标准的关系
- **ASME V&V 10/20/40**：计算固体力学/流体力学/医疗器械的V&V标准
- **IEEE 1597.1**：计算电磁学建模验证标准
- **OMB Circular A-119**：联邦标准制定政策
- **DoD和能源部V&V指南**
- **ASME V&V 40反向采用**了NASA-STD-7009A的若干概念

## 与MOSA/数字工程的关系

NASA-HDBK-7009A与数字工程的关系主要体现在：
- **M&S置信度框架**是数字孪生（digital twin）可信度评估的基础
- **验证/确认体系**为[[基于模型的系统工程]]中模型的V&V提供NASA级实践参考
- **全生命周期M&S管理**与[[数字工程生态]]中的数据治理要素对齐
- **NASA-STD-7009A**是NASA-HDBK-1009A（系统建模手册）引用的重要标准
- M&S工作表方法可扩展到MOSA接口仿真的质量保证
