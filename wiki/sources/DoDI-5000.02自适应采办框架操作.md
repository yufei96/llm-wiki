# DoDI 5000.02 — 自适应采办框架操作

## 概述

- **全称**：Operation of the Adaptive Acquisition Framework
- **发布方**：OUSD(A&S)，Ellen M. Lord 签署
- **生效日期**：2020年1月23日，Change 1 2022年6月8日（William A. LaPlante 批准）
- **取消/取代**：原 DoDI 5000.02 系列

## 目的

依据 DoDD 5135.02 授权，本指令：

1. 建立管理采办项目的**政策和程序**（依据 Title 10, U.S.C. 相关条款）
2. 分配采办项目管理职责
3. 描述主要负责人职责和各**采办路径**的目的与关键特征
4. 重组国防采办指导以提高流程效能，**实施自适应采办框架（AAF）**

## 六条采办路径

AAF 用六条采办路径取代了传统"一刀切"模式：

| 路径 | 对应指令 |
|------|---------|
| 重大能力收购（MCA） | DoDI 5000.85 |
| 中间层采办（MTA） | DoDI 5000.80 |
| 紧急能力采办（UCA） | DoDI 5000.81 |
| 软件采办（SWP） | DoDI 5000.87 |
| 国防业务系统（DBS） | DoDI 5000.75 |
| 国防服务采办（DAS） | DoDI 5000.74 |

## 关键政策要点

1. **裁剪原则**：项目经理和里程碑决策当局（MDA）可根据项目特点裁剪采办流程，不要求统一遵循所有要求
2. **职能领域**：覆盖系统工程、合同、测试评估、生命周期后勤、网络安全、知识产权等全部 10 个职能领域
3. **数字工程**：鼓励在整个生命周期中使用数字工程和模块化开放系统方法（MOSA）
4. **竞争性采办**：强调通过竞争降低成本和避免供应商锁定
5. **里程碑对齐**：每个路径有独立的里程碑结构和决策点，但在 AAF 框架下互通

## AAF 与 MOSA 的关系

DoDI 5000.02 是 MOSA 进入采办流程的**顶层入口**：
- 要求项目经理在采办策略中考虑 MOSA
- 为各路径（尤其 5000.85 MCA 和 5000.87 SWP）提供集成 MOSA 的政策授权
- 与 DoDI 5000.97（数字工程）互补，构成数字工程+开放架构的双支柱
