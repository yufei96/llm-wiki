# 迈入数字未知 — MBSE与数字工程演进

## 概述

INCOSE前主席（2014/2015）David Long在MBSE in Practice 2025上的主旨演讲"Into the Great Digital Unknown"。从系统复杂度爆炸的背景出发，系统阐述了数字工程的核心理念、MBSE旅程的经典陷阱、以及从"数字化"到"数字化转型"的本质区别。演讲融合了INCOSE SE Vision 2035的前瞻视角。

## 核心论点

### 1. 系统复杂度的范式转移

从静态产品到智能系统之系统（SoS）的演进路径：
```
机电产品 → 网络化产品 → 智能互联产品 → 产品体系 → 体系之系统
```

- 当代挑战：系统规模、任务复杂度、技术复杂度、团队复杂度、动态复杂度
- 传统工程方法（文档驱动）无法应对这种复杂度

### 2. 数字工程的核心理念

David Long提出四点原则：

- **显式优于隐式**（Explicit over Implicit）
- **清晰优于模糊**（Clarity over Ambiguity）
- **权威数据优于制品**（Authoritative Data over Artifacts）
- **计算、通信和访问**是数字工程的基础能力

数字工程vs传统工程的核心转变：从"基于文档"到"基于模型"不仅是工具替换，而是信息管理范式的根本变化。

### 3. 关键概念关系

```
系统工程 → 项目团队的技术结缔组织
数字工程 → 现代工程企业的关键使能器
数据      → 21世纪工程和运营的氧气
MBSE     → 数字工程环境的结缔组织
```

### 4. MBSE旅程的经典陷阱

- **卓越孤岛**（Silos of Excellence）：MBSE、ALM、MCAD、ECAD各自为政，互不连接
- **数字主线断裂**：概念→开发→生产阶段的信息断层
- **语言障碍**：不同生命周期阶段使用不同语言/符号——"什么是功能？"有不同答案
- **盲目追逐炒作**：需要评估和基准测试能力，而非买工具

### 5. 数字化转型的本质

- **Adoption ≠ Transformation**：采纳工具只是数字化，转型需要改变组织行为
- 数字孪生：连接设计与运营的"智能"虚拟表示
- 跨学科和整合思维：拥抱多样性视角，包容性、整体性、系统性思维

### 6. INCOSE SE Vision 2035

- 系统工程作为"更好明天的推进器"
- 从产品到体系之系统的演进
- 数字工程作为系统工程的新前沿
