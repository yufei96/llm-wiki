# NASA-HDBK-7009A — NASA 建模与仿真实施指南

## 概述

- **全称:** NASA Handbook for Models and Simulations: An Implementation Guide for NASA-STD-7009A
- **编号:** NASA-HDBK-7009A (Revision A)
- **发布方:** NASA Chief Engineer办公室（Ralph R. Roe, Jr. 签署）
- **批准日期:** 2019年5月8日
- **取代:** NASA-HDBK-7009 (Baseline, 2013年10月18日)
- **页数:** 157页
- **类型:** NASA Technical Handbook（指导性，非强制性标准）
- **配套标准:** NASA-STD-7009A (Standard for Models and Simulations)

## 定位与目的

HDBK-7009A是NASA-STD-7009A（建模与仿真标准）的**实施指南**。NASA-STD-7009A规定了M&S必须满足的**要求**（what），HDBK-7009A提供**实现方法**（how）。手册提供技术信息、应用指南、数据、推荐实践、程序和方法，帮助项目满足标准要求。

由于建模与仿真涵盖广泛的技术学科，并非所有方法同样适用于所有类型的M&S——手册强调需要根据具体的M&S类型进行**解释和裁切**。

## M&S生命周期过程模型

HDBK-7009A定义了NASA项目中M&S的完整生命周期，直接映射NASA系统工程项目生命周期阶段：

### Phase Pre-A: 模型启动

| 活动 | 内容 |
|------|------|
| **收集RWS信息** | 获取真实世界系统的数据和规格 |
| **M&S使用意图声明** | 明确M&S的预期用途和决策支持角色 |
| **关键性评估** | 评估M&S结果对决策的影响程度——决定后续V&V的严格程度 |
| **M&S方法论证** | 论证所选建模和仿真方法的适用性 |
| **经验数据可用性评估** | 评估可用于验证的经验数据 |

**产出:** M&S启动阶段产品和预期成果

### Phase A: 模型概念开发

- **RWS/环境规格和数据获取**：定义真实世界系统和环境的边界条件
- **模型概念权衡研究**：比较替代建模方法
- **初步模型需求和规格**：定义模型必须满足的功能和性能规格

### Phase B: 模型设计

- 实现模型设计
- 设计层面考虑因素
- **模型概念（设计）验证**：在设计阶段验证设计概念的合理性

### Phase C: 模型构建

- 实现模型构建
- 构建层面考虑因素

### Phase D: 模型测试与发布

这是V&V的核心阶段：

| 子阶段 | 关键活动 |
|--------|---------|
| **模型验证（Verification）** | 确认模型按照设计规格正确实现——"我们正确地构建了模型吗？" |
| **模型经验确认（Empirical Validation）** | 确认模型行为充分准确地代表真实世界系统——"我们构建了正确的模型吗？" |
| **模型发布（Release）** | 正式发布已验证和确认的模型供使用 |

### Phase E: 模型使用

- **使用前准备**：使用准备度评估
- **使用评估**：评估模型是否适合特定用例
- **输入场景定义和谱系**：定义和追踪输入数据的来源和质量

### 后续阶段

手册还涵盖模型的**维护、更新和退役**，形成完整的M&S生命周期闭环。

## 核心概念

### 模型的关键概念（Section 4.5）

- **谱系与溯源（Pedigree and Provenance）**：追踪模型及其数据的来源和历史——这是M&S可信度的基础
- **模型之模型（Model of Models）**：元模型概念，用于描述模型的组成和关系
- **NASA建模的动机**：为什么NASA依赖建模——包括不可测试场景、成本约束和安全要求

### M&S可信度

手册建立了M&S可信度框架，将可信度分解为多个维度，包括：

- **验证状态**：模型是否经过充分的验证和确认？
- **不确定性特征化**：M&S结果的不确定性是否被充分量化和传播？
- **适用性**：模型是否适合当前的预期用途？
- **谱系**：模型和数据的来源是否可追溯？

### 与NPR 7150.2的关系（Section 4.7）

HDBK-7009A与NASA软件工程要求（NPR 7150.2）对接，解决软件密集型模型中软件保证的特殊考虑。

### 项目管理和授权技术权威的职责（Section 4.8）

定义了项目经理和授权技术权威在M&S治理中的分工和职责。

## 三层手册架构中的位置（扩展）

NASA通过多层手册体系覆盖M&S治理：

| 文件 | 类型 | 作用 | 强制性 |
|------|------|------|--------|
| **NASA-STD-7009A** | Standard | M&S必须满足的要求 | 强制性（通过合同引用） |
| **NASA-HDBK-7009A** | Handbook | 如何满足要求的实施指南 | 指导性 |
| **NASA/SP-2016-6105 Rev 2** | Standard | 系统工程手册——引用M&S要求 | 强制性 |
| **NASA-HDBK-1004** | Handbook | 数字工程采办——引用M&S数据交付 | 指导性 |

## 对MOSA的意义

HDBK-7009A的M&S生命周期方法对MOSA具有多层面的适用性：

- **模块独立性验证**：M&S可信度评估框架可直接应用于MOSA模块的独立验证——每个模块作为独立的M&S对象，需要经过标准化的V&V流程
- **接口兼容性分析**：不确定性量化（UQ）方法可用于分析模块间接口的兼容性——当两个模块通过标准接口集成时，各自的不确定性如何传播和组合？
- **模块认证流程**：V&V流程与MOSA模块认证流程概念一致——Verification确认模块实现符合接口规格，Validation确认模块在实际使用环境中的表现
- **M&S生命周期映射**：NASA的Pre-A到Phase E生命周期为MOSA模块从概念到退役提供了成熟的过程模型模板
- **谱系与溯源**：在MOSA多供应商生态中，模型和数据的谱系溯源对于确保跨组织集成的一致性和可靠性至关重要
