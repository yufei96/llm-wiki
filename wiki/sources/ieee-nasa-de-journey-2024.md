# IEEE Aerospace Conference: NASA数字工程转型之旅（2024）

## 摘要

Terry R. Hill（NASA工程办公室数字工程项目经理）于2024年3月8日在IEEE航空航天大会（Big Sky, Montana）发表的演讲，全面展示了NASA数字工程转型的战略框架、目标、进展和挑战。这是了解NASA实际DE实施状态的一手资料，涵盖从FY2019到FY2024的完整演进历程。

## 要点

### NASA数字工程转型动机
NASA面临的核心驱动力：
- **任务越来越大胆复杂**（如Artemis深空探索）
- **合作伙伴越来越多**（国际合作、商业合作伙伴）
- **要求更快**（从概念到运行的时间压缩）
- **要求更可负担**（预算约束）
- **要求更透明**（公共问责）
- **要求更包容**（多元化参与）

### NASA DT战略框架（五层结构）
```
技术基础 → 数字杠杆 → 转型目标 → 目标 → 任务成果
```

| 层级 | 内容 |
|------|------|
| **技术基础** | 零信任架构、物联网、扩展现实、Model-Based Anything、AI/ML、智能自动化 |
| **数字杠杆** | 最大化数据影响、建立可互操作架构、转型关键流程、采用通用工具、加强包容性团队 |
| **转型目标** | 转型工程、转型发现、转型运行、转型决策制定 |
| **目标** | 大胆复杂任务、现代化工作方式、吸引留住人才、更敏捷流程、无缝伙伴合作、可持续运行 |
| **任务成果** | 更大胆任务、现代工作方式、吸引留住人才、敏捷流程、无缝伙伴合作、可持续运行 |

### DE需求、目标与目标体系

**核心需求**：通过整合流程、数字工具和技术，在整个工程系统开发生命周期中实现信息无缝流动，改善NASA工程领域的运作。

**7大目标**：
1. **G1 生命周期**：建立贯穿工程生命周期的DE战略
2. **G2 部署**：开发可互操作、可定制、可扩展的DEE部署策略
3. **G3 指导**：建立模型开发、工具集成和数据线索的指导
4. **G4 ASoT**：建立权威真实数据源的管理、治理、安全、追溯
5. **G5 配置/变更管理**：演进现有CM方法以支持数据驱动的工程基线管理
6. **G6 数字线程**：开发数字线程/生态系统策略
7. **G7 文化与劳动力**：演进NASA文化和劳动力

### MBSE能力评估（INCOSE工具）
NASA使用INCOSE MBSE能力评估工具在8个维度上评估各中心：

| 能力维度 | NASA平均(FY22) | NASA平均(FY23) | 趋势 |
|----------|---------------|---------------|------|
| 劳动力/文化 | 0.5 | 提升 | ↑ |
| SE流程 | 0.5 | 提升 | ↑ |
| 项目/计划流程 | 0.3 | 提升 | ↑ |
| 模型有效性 | 0.6 | 提升 | ↑ |
| IT基础设施 | **1.6** | 提升 | ↑（最高） |
| 建模工具构建 | 0.4 | 提升 | ↑ |
| 模型使用 | 0.4 | 提升 | ↑ |
| 建模政策 | 1.1 | 提升 | ↑ |

**关键发现**：IT基础设施得分最高（1.6），表明基础设施投入已领先；但SE流程和项目流程得分较低，说明文化变革滞后于工具部署。

### NPR SysML建模（数据驱动分析）
NASA跨中心合作建模了NPR 7120.5/.8、7123.1、NPD 7120.4、NPR 8705.2/8705.4：
- **860个Blocks**（各类数据、角色、类型等）
- **126个Activities**（生命周期评审的一部分）
- **460个Terms**（术语、角色、数据产品、生命周期、组织、运行领域、项目类型）
- **49处差异**（如数据交付目标不存在、跨NPR不一致）
- **25个分类标签**（成本、进度、需求等，2900+元素被标签化）

### MBSE工具部署状态（FY23末）
- **692个唯一用户**遍布所有中心
- **1100+个模型**在OCE Teamwork Cloud上创建和管理
- **5个任务局**使用（ARMD、ESDMD、SOMD、STMD、SMD）
- Agency级MagicDraw许可证服务器和Teamwork Cloud

### MBSE培训体系（5级能力模型）
| 级别 | 角色 | 核心技能 |
|------|------|---------|
| Tier 1 | 管理者/评审者 | 理解MBSE概念、读图、SE应用 |
| Tier 2 | LSE技术领导 | 基础设施建议、SysML文字素养、模板使用 |
| Tier 3 | 建模者 | 基本建模、导航他人模型、模式创建、规划遵循 |
| Tier 4 | 建模领导 | 配置文件/元模型创建、建模规划策略、集成模型 |
| Tier 5 | 专家建模者 | 脚本、模型集成、全栈MBSE |

### 工具链集成试点（Phased Integration）
分阶段集成NASA工程领域常用工具：
- **Phase I**：SysML建模→模型管理→ECAD PLM→SPDM
- **Phase II**：ET2TC（现有工具间连接器）、联邦IDP、云许可证管理、PLM、分析
- **Phase III**：GN&C软件与仿真、代码管理、MONTE/Trick/Post2等

### 设计、制造与检查能力定义
LaRC的数字线程制造（DTM）展示了MB数字线程从设计到制造的端到端集成：
- 机械设计→CNC/复合材料制造
- 电气设计→PCB制造
- 光学设计→增材制造
- **MB检查**：激光扫描→与设计模型对比→数字化不符合项输出

### 对工业界的呼吁
- **软件供应商**：与政府机构合作确定支持数字工具链的行业标准，并尽快合规
- **航空/国防工业伙伴**：与政府机构合作确定协作方式（工程和数据交换），同时保护IP

## 与MOSA/数字工程的关系

- NASA如何实际部署数字工程（不同于DoD的政策驱动，NASA是实践驱动）
- MBSE从志愿者行动到机构级部署的路径
