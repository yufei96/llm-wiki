# OSA合同指南 V1.1 — 开放系统架构合同语言

## 概述

国防部开放系统架构（OSA）合同指南，2013年5月发布，由OSA数据权利团队编制。**Frank Kendall**（时任采办、技术与后勤副部长，后任空军部长）签署发布备忘录。这是MOSA概念在合同层面最早的正式指南——在2017年NDAA $\S$805立法强制MOSA之前4年即已奠定合同基础。

## Frank Kendall签署备忘录要点

- 指南面向**全体采办人员**
- 目标：在招标书中嵌入OSA原则，增加竞争机会、改善创新获取
- 体现了**Better Buying Power**倡议的长期视角
- 补充FAR（联邦采办条例）和DFARS（国防FAR补充条例）
- **OSA与数据权利必须早期考虑**——在采办策略中给予适当权重
- 所有采办专业人员被指示**必须熟悉并实施本指南**

## 指南核心内容

### OSA合同语言类别

指南为采办专业人员提供**标准化合同条款模板**，覆盖：

1. **技术架构条款**：要求承包商交付开放架构设计、接口定义文档
2. **数据权利条款**：确保政府获得足够的接口数据权利以支持未来竞争
3. **模块化条款**：要求系统设计支持可分割组件的增量添加/移除/替换
4. **接口标准条款**：要求使用广泛支持的、基于共识的标准进行接口验证
5. **竞争条款**：确保在系统生命周期内多个供应商可竞争组件合同

### 合同语言设计原则

- **长期视角**：不仅关注当前采办，更确保未来升级/替换的竞争性
- **平衡风险与成本**：通过OSA降低生命周期成本
- **技术+商业双重维度**：OSA不仅是技术架构选择，更是商业模式选择

## 历史意义

| 时间 | 事件 |
|------|------|
| 2013年5月 | OSA合同指南V1.1发布（Frank Kendall备忘录） |
| 2016年12月 | FY2017 NDAA $\S$805签署——首次立法强制MOSA |
| 2019年12月 | FY2020 NDAA $\S$840——要求各军种发布MOSA实施指南 |
| 2020年12月 | FY2021 NDAA $\S$804——扩展至所有采办项目+接口存储库 |
| 2025年2月 | DoD MOSA实施指南手册发布 |

**OSA合同指南提前4年为MOSA立法提供了合同执行基础**——立法强制了"做什么"，指南提供了"怎么用合同实现"。

## 与MOSA知识库的关联

- 这是理解MOSA**合同执行维度**的关键文件——MOSA不仅是技术标准问题，更是合同语言和数据权利问题
- DFARS合同条款系列（第207/227/252部分）是本指南的法律基础
- GAO-25-106931指出的"缺乏MOSA成本效益分析"——根源在于合同层面未强制要求投资回报模型
