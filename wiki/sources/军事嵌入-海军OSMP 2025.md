# 海军将更多控制开放架构开发——OSMP内部化转型

## 概述

- **来源**: Military Embedded Systems，2025年8月27日报道
- **核心变化**: 美国海军正转变开放架构管理方式——**由海军内部撰写开放系统管理计划(OSMP)初稿**，而非将第一版交由承包商完成
- **关键人物**: Rich Ernst，高级开放架构主题专家(SME)
- **意义**: 代表海军MOSA实施从"外包文档"到"内生能力"的战略转变

## 旧模式 vs 新模式

| 维度 | 旧模式 | 新模式 |
|------|--------|--------|
| 文档起草 | 承包商起草OSMP第一版 | 海军内部撰写初版 |
| 执行方式 | "打勾练习"（合规负担） | 战略工具（架构蓝图） |
| 文档命运 | 常变成"架子文件"(shelfware) | 活的参考文件，指导全生命周期 |
| 优先级来源 | 承包商定义 | 海军从一开始嵌入自身优先级 |
| 架构决策 | 分散在各合同阶段 | 在项目启动时即捕获和固化 |

## OSMP核心目标

新OSMP的四项核心强调:
1. **模块化** (Modularity)
2. **标准合规** (Standards compliance)
3. **全寿命周期可负担性** (Lifecycle affordability)
4. **复用** (Reuse)

## Rich Ernst关键洞察

1. **"活的参考文件"**: OSMP将作为贯穿整个项目生命周期的动态参考，而非提交后即被遗忘的静态交付物
2. **"防止交易掉价值"**: 通过在项目启动时捕获架构决策，防止后续项目办公室因进度/成本压力而妥协架构完整性
3. **人员连续性保障**: 在平台维持数十年的过程中，人员必然变动——OSMP确保架构原则的连续性不因人而变
4. **接口识别与标准对齐**: 新方法帮助项目办公室识别关键接口、与演进的标准保持一致、利用研发成果吸纳新技术
5. **小企业机会创造**: 标准化分解和复用策略为小企业和非传统供应商创造了跨项目、跨军种的模块供应机会

## 战略对齐

该努力旨在构建支持以下目标的持久框架:
- 持续竞争
- 多元供应商生态
- 更具韧性的系统设计
- 完全对齐MOSA原则

## 笔记

1. **OSMP内部化是海军MOSA指南V1.0的逻辑延伸**: 指南明确指出"政府/获取方应在项目定义过程中起草OSMP，承包商以政府提供的OSMP为基础，仅做匹配其提案细节的微小调整"——本报道是该政策的具体执行信号。
2. **"架子文件"的根因分析**: OSMP外包给承包商之所以失败，不是因为承包商不称职，而是因为合同结构错误地将架构决策权外包。内部化纠正了这一根本性的采办激励错位。
3. **与DoD总体趋势一致**: 海军OSMP内部化与AFMC的政府参考架构(GRA)策略（Garrett演讲）方向一致——政府在架构层面从"审核者"变为"定义者"。
4. **小企业影响**: 标准化分解不仅降低供应商门槛，更创造了一个关键转变——模块可跨项目/跨军种供应，这对于依赖单一项目合同的小企业是重要的市场扩大机制。
5. **与项目管理的时间张力**: 内部化OSMP要求在项目启动阶段投入更多前期工程资源——这需要项目管理文化与采办节奏的支持，是该转型面临的主要实施挑战。
