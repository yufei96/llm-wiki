# ISO/IEC/IEEE 15288:2023 — 系统生命周期过程

## 来源

ISO/IEC/IEEE 15288:2023(E)，第二版，2023年5月发布。国际标准化组织（ISO）、国际电工委员会（IEC）、电气和电子工程师协会（IEEE）联合发布。

## 概述

系统和软件工程领域的**基础性国际标准**，定义了系统生命周期过程的框架。适用于各类系统的全生命周期——从概念到退役。这是系统工程的"宪法"级标准。

## 标准结构

ISO 15288定义了**四类过程组**：

### 1. 协议过程（Agreement Processes）
- 采办（Acquisition）
- 供应（Supply）

### 2. 组织项目使能过程（Organizational Project-Enabling Processes）
- 生命周期模型管理
- 基础设施管理
- 项目组合管理
- 人力资源管理
- 质量管理
- 知识管理

### 3. 技术管理过程（Technical Management Processes）
- 项目规划
- 项目评估与控制
- 决策管理
- 风险管理
- 配置管理
- 信息管理
- 度量

### 4. 技术过程（Technical Processes）
- 利益相关者需求定义
- 系统需求定义
- 架构设计
- 设计定义
- 系统分析
- 实施
- 集成
- 验证
- 确认
- 移交
- 运行
- 维护
- 退役

## 与MOSA的关系

ISO 15288为MOSA提供**过程框架基础**：

- **架构设计过程**：MOSA的模块化设计嵌入此过程
- **验证/确认过程**：MOSA的"一致性认证"原则需要ISO 15288的V&V过程支撑
- **利益相关者需求定义**：MOSA的开放接口需求来源于此
- **配置管理过程**：MOSA的接口版本管理需要配置管理支撑

MOSA是**设计策略**，ISO 15288是**过程框架**——MOSA在ISO 15288的架构设计过程中落地。

## 2023版更新要点

第二版（2023）相比第一版（2015）的关键更新：
- 更强调整体系统视角（holistic system perspective）
- 增加了敏捷和DevOps的适配
- 强化了信息安全过程
- 更新了与ISO 9001等质量管理标准的映射

## 与其他标准的关系

| 标准 | 关系 |
|------|------|
| ISO 9001 | 质量管理体系，ISO 15288的过程可满足ISO 9001要求 |
| EIA 632 | 系统工程过程标准，ISO 15288的前身之一 |
| IEEE 1220 | 系统工程过程标准，已被ISO 15288吸收 |
| ISO/IEC/IEEE 12207 | 软件生命周期过程，与15288并行 |
| INCOSE SE Handbook | INCOSE基于ISO 15288编写的SE实践指南 |

## 对知识库的价值

ISO 15288是知识库此前缺失的**SE过程框架基础**。之前MOSA讨论集中在"做什么"（设计策略）和"用什么"（标准/工具），但缺少"怎么做"（过程框架）。ISO 15288填补了这一层。
