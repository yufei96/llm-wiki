# Chris Garrett AFMC数字化转型演示（2024年9月）

## 摘要

这是空军装备司令部（AFMC）系统工程主管Chris Garrett（AFLCMC/EN）在2024年9月发表的演示文稿，主题为"USAF数字化转型：Think Big, Start Small, Scale Fast"。演示覆盖了空军生命周期管理中心（AFLCMC）的数字化物资管理（DMM）战略、政府参考架构（GRA）体系、MBSE模型库建设、以及数字化生态系统愿景。

演示以中美海军能力对比开场，展示美国海军作战优势持续被侵蚀的趋势，强调美国获取能力的速度远慢于中国——中国无需遵循DoD 5000流程和官僚程序要求。这构成了数字化转型的紧迫性背景。

## 要点

### 一、战略背景：美国能力获取速度劣势

- 中美海军能力列装趋势对比显示：中国多项关键能力（DF-21D反舰弹道导弹、J-15舰载战斗机、YJ-12/18反舰巡航导弹等）从初始估计到实际列装的时间远短于美国同类项目
- 美国项目（如F-35、SM-6、NIFC-CA、NGJ等）普遍延迟
- 核心原因：中国"获取设计——无DoD 5000——无官僚程序要求"

### 二、AFMC数字化战略倡议

**六大战略方向**：
1. 结构化和安全化数据
2. 促进AFMC范围内的数字培训
3. 促进DMM工具获取
4. 制定各AFMC组织的数字战略
5. 灌输数字优先文化
6. 现代化IT基础设施

**AQ数字采办优先事项**：
1. 实施开放系统标准和[[模块化架构模式|参考架构]]
2. 确保项目"天生数字化"或在生命周期内数字化适应
3. 扩展企业解决方案，拥抱基于云的协作环境
4. 制度化敏捷软件开发和软件密集型系统流程

### 三、数字化物资管理（DMM）终端状态

- **目标**：通过数字采办和物资管理加速能力交付
- **愿景**：DAF数字生态系统和充分赋能的数字劳动力，以前所未有的政企协作交付集成、创新和可信的能力
- **目的**：采办企业的速度、灵活性和敏捷性；更快交付制胜能力

**DMM六大支柱**：
1. 制定数字战略，摒弃陈旧实践
2. 灌输数字优先文化
3. 提供DMM工具访问
4. 现代化IT基础设施
5. 培训数字劳动力
6. 结构化和安全化数据

### 四、数字化挑战

- 企业资源配置（建立企业解决方案/方法）
- 劳动力教育、培训和采纳
- 工具互操作性和软件供应商锁定
- 软件测试、批准、部署流程
- 数据标准、格式、建模风格指南
- USAF vs USSF vs DAF集成和共性
- IT基础设施、工具、企业数字环境
- 跨项目、DAF和行业扩展企业解决方案
- 遗留项目和流程过渡
- 跨域解决方案/多级安全（CDS/MLS）可扩展性和标准化
- 数据项描述、合同数据需求清单、合同语言、IP/数据权利
- 实时政府/承包商数据访问 vs 传统时间分阶段交付

### 五、数字联盟协调领域

- 数据和模型管理、验证、确认（随时间）
- 跨项目、DoD和行业扩展数字工程企业解决方案
- 流程数字化（不仅是数字化现有流程，而是数字化新流程）
- 遗留工件、流程和最佳实践过渡
- 技术壁垒和政策障碍

### 六、政府参考架构（GRA）体系

**抽象模型分层管理**：
- 功能配置基线 → 分配配置基线 → 生产配置基线
- 从抽象到具体的模型层级

**AV GRA（飞行器政府参考架构）建设**：
1. 构建基于文档的参考模型库
   - 将MIL标准、手册等从PDF转换为模型
   - 聚焦电气子系统文档（MIL-HDBK-516第12节和JSSG-2009-8）
   - 保留原始文档内容但扩展模型元素
2. 构建AV GRA
   - 基于MIL-STD-881的AV结构
   - 通用内容：逻辑架构、需求、验证等
3. 使用敏捷方法管理（Jira看板和Sprint）

**参考模型库详情**：
- 79个文档 → 36个系统模型
- 聚焦MIL-HDBK-516第12节（电气系统）和JSSG-2009-8引用的文档
- 包含：EPS组件（电池、发电机、转换器、逆变器、连接器等）、JSSG系列、其他MIL标准
- 模型表示保留文档结构，增加内容，应用通用实践
- VV&A流程和模型、样式指南、JSON格式

**功能GRA**：
- 武器GRA、传感器GRA
- [[WOSA架构工具|WOSA]]、CFW ICD、COARPS、[[开放任务系统|OMS/UCI]]、[[FACE技术标准|FACE]]、SOSA H/W
- PNT GRA：R-EGI、OMS/UCI、FACE、SOSA H/W

**目标架构**：
- 使命特定权衡，识别公共服务
- 软件应用服务
- 可复用共享资产：模型、服务、组件、软件应用
- 领域映射：EW→Big Iron/SOSA、Sensing→COARPS/SOSA、Comms→SOSA、Autonomy→A-GRA/SOSA、PNT→R-EGI/SOSA
- OMS作为隔离器，FACE用于航空电子，UAI用于武器

### 七、数字化生态系统

- **使能平台**：PLM / FENCES / CLOUDONE / PLATFORMONE
- **组成**：参考架构模型、标准、工具、培训、数据架构
- **企业作战分析**：HAF → AF FUTURES → MAJCOMS → 空军设计 → 所需能力 → 差距/机会 → AFWIC → 力量设计 → S&T需求/先锋/实验活动 → MBSE系统架构
- **未来状态**：采办方RFP → 供应商响应，通过数字化全生命周期管理

## 笔记

- DMM是AFMC层面的数字化转型框架，与DoD层面的数字工程生态系统（DEE）互补但视角不同
- GRA体系是MOSA在架构层面的具体实施：政府拥有参考架构，定义模块边界和接口
- 79个MIL文档→36个系统模型的转换是数字工程从文档驱动到模型驱动的典型案例
- "软件供应商锁定"被明确列为数字化挑战，直接呼应MOSA的反锁定目标
- 中国"无DoD 5000流程"的对比突出了美国采办流程对速度的制约，与[[速度与模块化权衡]]张力一致
- AV GRA基于MIL-STD-881构建，与[[MIL-STD-881D国防物资工作分解结构]]直接关联
- SOSA H/W在目标架构中作为硬件标准化层覆盖多个领域（EW、Sensing、Comms、Autonomy、PNT）
- 分布声明A，经PAIRs案例AFLCMC 2024-0203批准公开发布
