# 三本SE手册对比：NASA SP-2016-6105 vs INCOSE V5 vs ISO 15288

## 概述

知识库现有三本系统工程权威参考，形成标准→最佳实践→机构实现的完整层次。本文逐维度对比三者差异，为MOSA实施提供选择指南。

## 基本信息

```
                   NASA SE Handbook      INCOSE SE Handbook V5    ISO/IEC/IEEE 15288
编号               SP-2016-6105 Rev 2    INCOSE-TP-2003-002-05    ISO/IEC/IEEE 15288:2023
发布               2016                  2023 (Wiley)             2023
页数               ~260                  ~700                     ~80
定位               NASA机构实现          国际最佳实践              国际标准
语言               英文                  英文                      英文
对齐标准           NPR 7123.1            ISO 15288                 本身就是标准
```

## 维度对比

### 1. 生命周期模型

```
NASA:     Pre-Phase A → A → B → C → D → E → F
          (概念研究 → 概念开发 → 初步设计 → 最终设计 → 集成测试 → 运行 → 收尾)

INCOSE:   14个技术过程（非阶段式）
          任务分析 → 需求定义 → 架构定义 → 设计定义 → 实现 → 集成 → 验证 → ...

ISO:      与INCOSE对齐（INCOSE引用ISO 15288定义）
          同样是过程式，非阶段式
```

差异：NASA是**阶段门**模型（Phase A-F），INCOSE/ISO是**过程**模型。MOSA采办更接近NASA阶段门，但接口管理可参考INCOSE过程。

### 2. 架构定义

```
NASA:     未单独成章，融入SE引擎各阶段
          架构在Phase A/B定义，通过迭代细化

INCOSE:   独立过程（Section 2.3.5.4）
          明确列出架构质量属性：
          - modularity, openness, scalability
          - deployability, degree of automation
          - availability, reliability, affordability

ISO:      定义过程框架（输入/输出/活动）
          不规定具体质量属性
```

关键发现：**INCOSE V5 明确将 modularity 和 openness 列为架构质量属性**——这是三本手册中最直接的MOSA支持。

### 3. 接口管理

```
NASA:     独立章节（Section 6.3），最详细
          - 文档族：IRD → ICD → IDD → ICP
          - 接口工作组（IWG）
          - 物理/电气/机械/人机接口特征
          - 变更控制流程

INCOSE:   融入集成过程（Section 2.3.5.8）
          - 关注接口正确性验证
          - 物理/逻辑/人机接口
          - 无独立文档体系

ISO:      在集成过程中隐含
          - 定义"合成系统元素并验证接口"
          - 无具体方法论
```

差异：NASA最具体（有ICD模板），INCOSE次之（有方法论），ISO最抽象（只有过程定义）。

### 4. 韧性

```
NASA:     通过可靠性和安全性过程覆盖
          无独立韧性章节

INCOSE:   独立章节（Section 3.1.9）
          定义：面对逆境时提供所需能力
          三目标：避免/承受/恢复
          范围：所有逆境来源

ISO:      未单独讨论
```

差异：只有INCOSE将韧性提升为独立质量特性。NIST SP 800-160 Vol. 2 是INCOSE韧性的赛博特化版本（增加第四目标"适应"）。

### 5. MBSE/数字工程

```
NASA:     未深入
          提及模型但未讨论MBSE方法论

INCOSE:   专节讨论
          MBSE方法采用持续增长
          与数字工程方法衔接
          数字工程中数据透明、跨领域协作

ISO:      未讨论
```

差异：只有INCOSE讨论MBSE和数字工程。

### 6. 适用场景

```
NASA:     NASA内部项目
          - 有具体模板和流程
          - 有SE引擎框架
          - 适合NASA承包商

INCOSE:   通用系统工程项目
          - 最佳实践指南
          - 适合培训和参考
          - 适合MOSA架构设计

ISO:      合同和采办引用
          - 标准化过程定义
          - 适合MOSA采办合同
          - 适合审计和合规
```

## 对MOSA实施的指导意义

```
采办合同层：引用 ISO 15288 作为过程标准
架构设计层：参考 INCOSE V5 的 modularity/openness 质量属性
接口管理层：采用 NASA 的 ICD/IWG 工程实践
韧性设计层：结合 INCOSE 韧性框架 + NIST 赛博弹性原则
MBSE实施层：参考 INCOSE V5 的 MBSE 方法论 + SysML v2
```

## 关键发现

1. **INCOSE V5 是MOSA架构设计的最佳来源**：它明确将 modularity 和 openness 列为架构质量属性，且与 ISO 15288 直接对齐

2. **NASA 的接口管理最具体**：ICD 文档族和 IWG 工作组机制可直接用于MOSA接口管理实践

3. **三者不是替代关系而是互补关系**：ISO 给合同官员、INCOSE 给系统工程师、NASA 给工程团队——MOSA实施需要三者结合

4. **韧性是INCOSE V5的新增亮点**：三本手册中只有INCOSE将韧性提升为独立质量特性，这为MOSA的弹性设计提供了理论基础
