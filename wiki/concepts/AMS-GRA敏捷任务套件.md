# 敏捷任务套件政府参考架构（AMS GRA）

## 摘要

敏捷任务套件政府参考架构（Agile Mission Suite Government Reference Architecture, AMS GRA）是美国空军主导的政府拥有模块化开放系统框架，为NGAD（下一代空中优势）体系（Family of Systems）提供任务系统的架构标准。AMS GRA在开放架构四层体系中位于**设备级（Equipment/Unit）**，是2024年12月三军联合备忘录确认的6大已验证MOSA标准之一。其核心理念为 **"NGAD FoS is Software Defined and Hardware Enabled"**——通过软件与硬件解耦，实现任务系统在动态威胁环境中的快速适应。

## 核心目标

1. **软件硬件解耦**：将任务系统软件与飞行平台硬件分离，实现独立迭代和升级
2. **竞争性采购**：通过政府拥有的开放架构，建立软件和硬件供应商"市场"
3. **快速能力部署**：通过CI/CD流水线（开发、集成、测试、验证）实现软件定义的能力更新
4. **跨平台复用**：为NGAD FoS家族中的有人/无人平台提供通用任务系统架构

## 核心特点

### 软件定义与硬件使能

AMS GRA的设计哲学明确区分：
- **软件定义层**：任务系统功能通过软件实现，支持动态重编程而无需更换硬件
- **硬件使能层**：传感器处理和平台处理通过SOSA兼容硬件基础设施提供支撑
- **"Sensor-to-Glass"**：政府拥有的从传感器到显示的全链路架构

### 开放接口与竞争生态

- **接口开放通用**：不依赖供应商锁定，所有接口为开放和通用标准
- **"市场"模式**：建立软件和硬件供应商的市场竞争环境
- **最小化适航影响**：新能力部署时对适航认证的影响降至最低

### 商业化与军用结合

AMS GRA采用双轨方法：
- **商业化方法**：利用商用容器化、数字孪生等技术降低成本、加速能力交付
- **军用化方法**：平台处理和传感器处理针对军用环境加固

### CI/CD流水线

- 开发、集成、测试、验证的持续集成/持续交付流水线
- 支持快速迭代和增量式能力发布

## 在GRA目标架构中的位置

AMS GRA位于开放架构四层体系的**设备级**，与SOSA、A-GRA等设备级标准横向协作：

```
武器(WOSA)  传感(COARPS)  通信    自主(A-GRA)  PNT(R-EGI)
    │           │          │         │           │
    └───────────┴──────────┴─────────┴───────────┘
                          │
              OMS（ASB隔离器）
                          │
                    ┌─────┴─────┐
                    │  FACE     │
                    │  SOSA H/W │  ← AMS GRA的硬件基座
                    │  CMOSS    │
                    └───────────┘
```

AMS GRA在架构层次中：
- **上层**：通过OMS/UCI与平台级（如ABMS GRA）集成
- **同层**：与A-GRA（自主）、COARPS（雷达）、R-EGI（PNT）等设备级GRA协作
- **下层**：基于SOSA硬件标准和FACE软件标准

## 与CCA的关系

AMS GRA是CCA（协同作战飞机）项目的核心政府参考架构之一：

- **CCA增量1**：AMS GRA与A-GRA作为两大政府参考架构，支撑CCA的采办策略（参见Andrew Hunter 2024年参议院证词）
- **CCA增量2**：AMS GRA被设计为国际合作的核心架构框架，盟国和合作伙伴可在AMS GRA标准化框架下贡献子系统
- **合作领域**：软件开发、传感器集成、任务系统等
- **战略意义**：通过MOSA实现技术互操作性，支持盟国间自主系统控制权的无缝交接

CSIS报告指出，盟国应**尽早熟悉AMS GRA标准**，为CCA增量2的国际合作做准备。

## 与其他标准的关系

| 关系类型 | 相关标准 | 说明 |
|----------|----------|------|
| **垂直依赖** | OMS/UCI | AMS GRA通过OMS抽象服务总线与平台级系统集成 |
| **硬件基座** | SOSA | AMS GRA的传感器和处理硬件基于SOSA标准 |
| **软件层** | FACE | 航空电子软件组件可移植性标准 |
| **横向协作** | A-GRA | 自主系统标准，与AMS GRA共同支撑CCA |
| **横向协作** | WOSA | 武器系统标准，通过UAI/CFWI与AMS GRA连接 |
| **横向协作** | COARPS | 雷达开放架构标准 |
| **三军确认** | 6大验证标准 | 2024年12月三军联合备忘录确认AMS GRA为6大已验证MOSA标准之一 |

## 应用案例

### Northrop Grumman NG InSight处理器

NG InSight是符合AMS-GRA的开放系统处理器实例：
- **合规标准**：OMS UCI 2.0 + 3U OpenVPX + SOSA + AMS-GRA
- **尺寸**：16" × 15" × 8"（45磅，850W）
- **能力**：视线通信（MADL/Link 16/DLOS）、超视距通信、SIGINT、网络攻防、AI/ML编排
- **设计特点**：数字孪生和容器化降低新能力成本并加速交付；液冷和风冷双版本适应不同平台
- **适用平台**：空、地、海、天多域环境

### NGAD FoS采办策略

AMS GRA在DAF下一代采办模式中扮演关键角色（Lt Gen Donna Shipton, AFLCMC/CC, 2024.8 DAU演示）：
- 从"黑盒子系统集成在供应商锁定的OFP中"向"开放子系统集成在开放政府参考架构中"演进
- 建立S&T到平台项目的桥梁，GRAs和标准为社区提供设计参数，加速技术转化

### CCA国际合作框架

CSIS分析（2024.12）指出AMS GRA在国际合作中的角色：
- 为CCA增量2的盟国参与提供标准化框架
- 简化出口能力设计——可通过模块替换移除含敏感技术的组件
- 支持建立"生产网络"（production web），各国在MOSA框架下专注于各自优势领域

## 笔记

- AMS GRA是2024年12月三军联合备忘录新增确认的两个标准之一（另一个是VICTORY），说明其在MOSA标准体系中的地位正在制度化
- "Software Defined and Hardware Enabled"的表述呼应了商业IT领域"软件定义一切"的趋势，但在军事领域需要同时解决适航认证和安全关键系统的约束
- AMS GRA的"Sensor-to-Glass"政府拥有概念意味着政府掌握了从传感器数据采集到操作员界面的完整架构链——这在传统模式下通常被主承包商锁定
- 与A-GRA的分工：AMS GRA关注任务系统（传感器、处理、通信），A-GRA关注自主算法和决策——两者共同构成CCA的"大脑"和"神经系统"
- NG InSight处理器的多标准合规（OMS UCI 2.0 + SOSA + AMS-GRA）展示了GRA体系中各标准的协同关系
- CSIS报告特别强调盟国应"now"就熟悉AMS GRA，暗示CCA增量2国际合作的时间窗口较紧
