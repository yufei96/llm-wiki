# SysML v1 到 v2 过渡指南

## 背景

OMG 于 2023 年 6 月通过 SysML v2 Beta 规范，2025 年 9 月发布正式规范。SysML v2 代表下一代系统建模语言，相比 v1 在精度、表达力、一致性、可用性、互操作性和可扩展性方面有显著提升。

## 三个核心规范

1. **KerML (Kernel Modeling Language) v1.0** — SysML v2 的元模型基础，基于一阶逻辑和四维时空范围定义形式语义
2. **SysML v2.0** — 复杂系统建模能力，提供互补的文本和图形表示
3. **SysML API & Services v1.0** — 标准服务接口，支持模型导航、查询和更新，实现工具互操作

## 关键术语

| 术语 | 定义 |
|------|------|
| 模型转换 (Model Conversion) | 将 SysML v1 基线模型转换为 v2 基线模型的完整过程 |
| 变换 (Transform) | 模型转换中的一个步骤，将 v1 模型翻译为 v2 模型 |
| 过渡 (Transition) | 组织或项目将 v1 建模环境、实践、培训更新为 v2 的过程 |

## 过渡指南产品

- **58 条 FAQ** — 来自 2023 INCOSE 国际研讨会的社区输入
- **过渡计划大纲** — 从 v1 过渡到 v2 需要考虑的规划要素
- **工具评估清单** — 基于模型转换经验的工具功能检查点
- **建模环境搭建** — SysML v2 文本符号开发环境配置指南
- **模型转换流程** — 模型转换的示例方法

## 与 MOSA/开放架构的关系

SysML v2 的标准化 API 和服务接口与模块化开放系统方法（MOSA）理念一致——通过标准化接口实现工具和模型的互操作，避免供应商锁定。
