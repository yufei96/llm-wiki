# 国防采办项目成本超支趋势

## 概述

美国国防部重大国防采办项目（MDAP）长期存在严重的成本超支和进度延误问题。Shah（2021）博士论文基于86个MDAP项目的历史数据进行了迄今最系统的定量分析——这一危机是推动 MOSA 政策出台的核心动因之一，也是理解 DoD 为何将"采用模块化设计"提升为**作战必要条件**的关键背景。

## 量化数据

### Shah 2021（86个MDAP，截至2019年）

| 指标 | 2010年前启动项目 | 2010年后启动项目 |
|------|------------------|------------------|
| 项目数 | 61个 | 25个 |
| 平均成本上涨 | **62.4%** | 2.1% |
| 总超支金额 | **5,421亿美元** | 53亿美元 |
| 平均延期 | **35个月** | 9个月 |

> 注：2010年后项目多数仍处于早期生产阶段，趋势可能随时间恶化。

### GAO 2024（108个项目）

| 指标 | 数据 |
|------|------|
| 审查项目总估值 | >$2万亿 |
| 未交付能力的MDAP平均周期 | **>10年** |
| 已交付能力的MDAP平均周期 | 11年（比原始计划延迟3年） |
| 传统MDAP项目平均成本增长 | **28%** |
| MTA项目平均成本增长 | 3% |
| 年度成本变化（31个MDAP） | -$17亿（主要来自数量削减和通胀假设调整） |

## 其他损失维度

- **技术过时**：每年造成约7.5亿美元损失（武器系统技术生命周期远超商用迭代周期）
- **网络威胁**：全球每年约6,000亿美元损失，约占全球GDP的1%
- **超60%的MDAP**存在超预算或延期问题
- **软件人才**：53个软件密集型项目中多数报告招聘留用困难；DoD软件干部队伍截至2024年3月仅1名联邦雇员

## 成因分析

1. **需求蔓延**：项目执行期间不断新增功能需求，缺乏模块化边界导致变更成本呈指数增长
2. **估算偏差**：初始成本和进度估算系统性偏低，部分源于"先报价低、再追加预算"的激励错位
3. **供应商锁定**：单一供应商依赖导致竞争缺失，后续合同缺乏议价能力（详见[[供应商锁定]]）
4. **技术生命周期错配**：国防项目周期（10-20年）远超商用技术迭代周期，系统部署时已过时
5. **瀑布式开发惯性**：GAO 2024指出"缓慢的线性开发方式仍在延续"——尽管2020年推出了自适应采办框架，多数项目仍未实施领先产品开发实践

## MOSA 的制度回应

MOSA 通过模块化架构和开放接口直接针对每条成因：

| 成因 | MOSA 对策 | 机制 |
|------|----------|------|
| 需求蔓延 | 模块化设计 | 变更限制在模块边界内，不波及全系统 |
| 供应商锁定 | 开放接口 | 竞争性重新采购，模块级多家供应商 |
| 生命周期错配 | 技术刷新 | 单独更换过时模块，不重建整个系统 |
| 瀑布式开发 | MTA + 模块化合同 | 5年快速原型 + 软件工厂加速迭代 |

## 进展与挑战

Shah（2021）的核心结论：**MOSA是方法而非结果**——定性分析表明使用 MOSA 策略的项目表现正在改善，但"缺少数据阻碍了统计检验"。评估什么样才算"成功"——用什么指标衡量、如何测量——仍然是关键挑战。

GAO 2024 确认了类似困境：MTA 路径项目成本增长远低于传统 MDAP（3% vs 28%），但多数 MTA 项目未规划实施领先实践来实现快速交付目标，计划转入 MDAP 的 MTA 项目平均仍需**10年**才交付初始能力。
